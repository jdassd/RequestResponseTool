# Request Response Tool

一款功能强大且轻量级的 Chrome 浏览器扩展程序，旨在帮助开发者、测试人员和安全研究人员掌控、拦截并改写网络请求与响应。本项目对标 Requestly 核心功能，提供极简且高效的本地调试体验。

## 🚀 核心功能

### 1. 请求控制 (Traffic Shaping)
- **重定向 (Redirect)**:
  - **完全重定向**: 将 URL A 的请求转发至 URL B（支持本地资源替换）。
  - **字符串替换**: 傻瓜式替换 URL 中的片段（如 `prod` 换成 `dev`），自动编译为高效正则执行。
- **取消/屏蔽 (Block)**: 阻止特定请求的加载，减少干扰或测试断网情况。
- **请求方法过滤**: 规则可精确限定于 `GET`、`POST`、`PUT`、`DELETE` 等特定 HTTP 方法。

### 2. Header 修改 (Header Modification)
- **自定义 Headers**: 动态添加、修改或删除请求头和响应头。
- **User Agent 伪装**: 内置常用设备预设（iPhone, Android, Desktop），一键切换浏览器身份。
- **自动补全**: 支持常用 HTTP Header 键名的自动补全建议。

### 3. 模拟与 Mock (Mocking & Rewriting)
- **Mock 响应**: 拦截请求并直接返回自定义的 JSON/文本内容。
- **网络延时 (Delay)**: 支持设置毫秒级延时，模拟弱网环境，测试前端 Loading 状态。
- **动态模板变量**: 响应体支持占位符，实现“伪动态”Mock：
  - `{{query.key}}`: 引用 URL 参数。
  - `{{time.now}}`: 当前时间戳。
  - `{{random.uuid}}`: 随机 UUID。
  - `{{request.url}}`: 原始请求地址。
- **高级拦截 (Intercept)**: 注入页面上下文，精细改写 `fetch` 和 `XHR` 的请求体与响应内容。

### 4. 脚本注入 (Script Injection)
- **JS/CSS 注入**: 在页面加载的不同阶段（Document Start / Ready / Idle）自动注入自定义脚本或样式。

### 5. 规则管理 (Management)
- **规则分组 (Groups)**: 支持创建分组，并可一键开启/关闭整个分组内的所有规则。
- **拖拽排序**: 自由调整规则优先级。
- **本地存储**: 所有配置均存储在浏览器本地，安全隐私。

### 6. 辅助工具
- **批量请求 (Batch)**: 导入 CSV/JSON/Excel，根据模板并发发送大量请求并查看统计结果。
- **命中日志 (Logs)**: 实时记录规则命中详情，方便回溯调试。
- **导入/导出**: 支持配置文件的 JSON 导入与导出。

## 🛠️ 安装与使用
1. 下载本项目代码。
2. 打开 Chrome 浏览器，进入 `chrome://extensions/`。
3. 开启右上角的“开发者模式”。
4. 点击“加载已解压的扩展程序”，选择本项目根目录。
5. 点击插件图标打开控制面板开始配置。
